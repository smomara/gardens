Favorite Programming Books

I feel inclined to preface this page with a precaution: These books should serve as necessary supplements to your programming practice - they are a means to an end, not an end within themselves.

Learning to code is done by coding. Do the exercises. Build CLI tools. Build simple HTTP servers. Build tools you want to use. Use open-source software, engage in its community, and read the code.

  1. Structure and Interpretation of Computer Programs: Teaches core programming concepts using Scheme. A seminal textbook in computer science education.

  2. Computer Systems: A Programmer's Perspective (CSAPP): Everything a systems programmer should know about how computers work. I don't think you can call yourself a serious programmer unless you understand most of this book.

  3. Crafting Interpreters: Practical guide to writing an interpreter, a great way to apply a bunch of programming techniques at once.

  4. Parallel and Concurrent Programming in Haskell: Showcases the beauty of Haskell. Once you learn about STM, you may never be able to go back.
